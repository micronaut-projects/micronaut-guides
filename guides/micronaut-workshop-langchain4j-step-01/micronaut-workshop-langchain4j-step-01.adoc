# Step 01 - Introduction to Micronaut LangChain4j

To get started, make sure you use the `step-01` directory.

This step is the starting point for the workshop.
It's a simple Micronaut application that uses the https://micronaut-projects.github.io/micronaut-langchain4j/latest/guide/[Micronaut LangChain4j] extension to interact with OpenAI's gpt-4o model.
It's a simple chatbot that we will extend in the subsequent steps.

## OpenAI API key

export your OpenAI key as an environment variable:

[source,bash]
----
export OPENAI_API_KEY=your_openai_api_key
----

common:runapp.adoc[]

NOTE: "Could not resolve placeholder ${OPENAI_API_KEY}". If you run into an error indicating `Error starting Micronaut server: Bean definition [io.micronaut.langchain4j.openai.DefaultOpenAiModerationModelConfiguration] could not be loaded: Error instantiating bean of type  [io.micronaut.langchain4j.openai.DefaultOpenAiModerationModelConfiguration] Message: Could not resolve placeholder ${OPENAI_API_KEY}`, make sure you have set the environment variable `OPENAI_API_KEY` with your OpenAI API key.

This will bring up the page at http://localhost:8080[http://localhost:8080].
Open it and click the red robot icon in the bottom right corner to start chatting with the chatbot.

## Chatting with the chatbot

The chatbot is calling gpt-4o (from OpenAI) via the backend.
You can test it out and observe that it has memory.
Example:

```
User: My name is Clement.
AI: Hi Clement, nice to meet you.
User: What is my name?
AI: Your name is Clement.
```

image::langchain4jworkshop/ui.png[]

A very important aspect of the interaction with LLMs is their statelessness.
To build a conversation, you need to _resend_ the full list of messages exchanged so far.
That list includes both the user and the assistant messages.
This is how the memory is built up and how the LLM can provide contextually relevant responses.
We will see how to manage this in the subsequent steps.

## Anatomy of the application

Before going further, let's take a look at the code.

If you open the build file, you will see that the project is a Micronaut application with the https://micronaut-projects.github.io/micronaut-langchain4j/latest/guide/#openai[`micronau-langchain4j-openai`] extension.

dependency:micronaut-langchain4j-openai[groupId=io.micronaut.langchain4j]

https://micronaut-projects.github.io/micronaut-langchain4j/latest/guide/#openai[Microanut LangChain4j OpenAI] is a dependency that provides a simple way to interact with language models (LLMs), like https://platform.openai.com/docs/models/gpt-4o[gpt-4o from OpenAI].
It actually can interact with any model serving the OpenAI API (like https://docs.vllm.ai/en/latest/[vLLM] or https://podman-desktop.io/docs/ai-lab[Podman AI Lab].
Micronaut LangChain4j abstracts the complexity of calling the model and provides a simple API to interact with it.

In our case, the application is a simple chatbot.
It uses a _WebSocket_, this is why you can also see the following dependency in the build file:
you need something

dependency:micronaut-websocket[groupId=io.micronaut]


If you now open the `src/main/java/example/micronaut/CustomerSupportAgentWebSocket.java`  file, you can see how the web socket is implemented:

source:CustomerSupportAgentWebSocket[]

Basically, it:

1. Welcomes the user when the connection is opened
2. Calls the `chat` method of the `CustomerSupportAgent` class when a message is received and sends the result back to the user (via the web socket).

Let's now look at the cornerstone of the application, the `CustomerSupportAgent` interface.

source:CustomerSupportAgent[]

This interface is annotated with `@AiService` to indicate that it is an AI service.
An https://docs.langchain4j.dev/tutorials/ai-services/[_AI service_] is an object managed by the Micronaut LangChain4j extension.
It models the interaction with the AI model.
As you can see it's an interface, not a concrete class, so you don't need to implement anything (thanks Micronaut!).
Micronaut LangChain4j will provide an implementation for you at build time.
Thus, your application only interacts with the methods defined in the interface.

There is a single method in this interface, `chat`, but you could name the method whatever you wanted.
It takes a user message as input (as it's the only parameter, we consider it to be the user message) and returns the response from the AI model.
How this is done is abstracted away by Micronaut LangChain4j.
